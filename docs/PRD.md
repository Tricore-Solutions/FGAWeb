# Product Requirements Document (PRD)  
_For Future Generation Academy Website_

---

## 1. Executive Summary
This PRD defines all system features, behaviors, and workflows for the FGA website project.

---

## 2. Goals & Deliverables

### Goals
- Full digital platform for FGA operations  
- Event, tournament, and program management  
- Membership accounts and subscription payments  
- Admin dashboard for staff  

### Deliverables
1. Dynamic website  
2. Registration system  
3. Events & tournaments module  
4. Membership accounts  
5. Subscription payment gateway  
6. Admin portal  

---

## 3. Functional Requirements

---

### A. Public Modules

#### Home Page
- Dynamic hero  
- Event highlights  
- CTA buttons  

#### About Page  
- History, mission, achievements  

#### Programs Page  
- Program list  
- Registration CTA  

#### Events & Tournaments
- Event listing  
- Calendar view  
- Event detail page  

#### Registration System  
Includes:  
- Program/tournament registration  
- Email confirmation  
- Optional WhatsApp integration  

Admin:  
- View registrations  
- Export data  

---

### B. Membership System  

#### User Accounts
- Signup/login/logout  
- Password recovery  
- Profile updates  

#### Subscription Payments
- Stripe checkout  
- Email receipts  
- Admin visibility  

---

### C. Admin Panel
- Dashboard  
- Event management  
- Registration management  
- Membership management  
- Payment records  
- Content updates  

---

## 4. Non-Functional Requirements

### Performance
- Page loads < 2 seconds  
- API < 400 ms  

### Security
- Password encryption  
- HTTPS  
- SQL injection & XSS protection  

### Scalability
- Prepared for mobile app integration  
- Expandable DB schema  

### Reliability
- 99% uptime (hosting dependent)

---

## 5. Integrations
- Stripe  
- Email service  
- Optional WhatsApp API  
- Hosting environments  

---

## 6. Technical Constraints
- Domain must be purchased by client  
- Client covers third-party fees  
- Stripe pricing and approval may vary  

---

## 7. Project Timeline (8–9 Weeks)
- UI/UX (Week 2–3)  
- Backend systems (Week 3–5)  
- Payment gateway (Week 5–6)  
- Admin panel (Week 6–7)  
- QA (Week 7–8)  
- Launch prep (Week 8–9)

---

## 8. Acceptance Criteria
- All modules functional  
- Admin panel fully working  
- Stripe integration complete  
- All pages responsive  
- Passed QA testing  

---

## 9. Risks & Mitigation

| Risk | Mitigation |
|------|------------|
| Late content submission | Request materials early |
| Gateway approval delays | Start setup early |
| Scope creep | Additional features billed separately |
| WhatsApp API pricing | Client informed early |

---

## 10. Post-Launch Support
- 6 months free bug fixes & optimizations  
- New features require new quotation
